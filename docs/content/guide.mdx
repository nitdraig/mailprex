# User fields guide

The `POST /email/send` endpoint accepts JSON and delivers a formatted email to the account owner.

**Base URL:** `https://api.mailprex.excelso.xyz/email/send`

## Required fields

| Field | Type | Description |
|---|---|---|
| `emailDestiny` | string | Recipient email — **must match** the Mailprex account that owns the token |
| `fullname` | string | Name entered in the form |
| `email` | string | Email entered in the form |
| `message` | string | Message body |
| `webName` | string | Website or form label (used in subject line) |
| `formToken` | string | Private token from the dashboard (`mk_live_…`) |

## Optional fields

| Field | Type | Description |
|---|---|---|
| `phone` | string | Sender phone number |
| `service` | string | Service or topic |
| `captchaToken` | string | Cloudflare Turnstile token (required when CAPTCHA is enabled on the API) |

## Validation and limits

- Invalid or missing token → `401`
- `emailDestiny` ≠ account email → `403`
- Monthly quota exceeded → `429`
- Too many sends to the same recipient per hour → `429`

Plan limits: Free 200/month, Pro 5,000/month, Business unlimited.

## Token security

- New tokens use prefix `mk_live_` and are stored as a bcrypt hash server-side.
- The dashboard shows only a prefix after generation; regenerate if you lose the secret.
- Legacy UUID tokens still work until regenerated.

See [Migration v2](/migration-v2).
